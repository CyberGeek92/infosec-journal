Overview

Gobuster is a fast, command-line brute-forcing tool written in Go. It's used to discover hidden directories, files, DNS subdomains, and virtual hosts on web servers. Unlike tools like Dirb or Dirbuster, Gobuster is optimized for speed and minimal resource usage.

Core Capabilities

  • Directory and file brute-forcing
  • DNS subdomain enumeration
  • Virtual host discovery
  • S3 bucket enumeration
  • Custom HTTP headers and status filtering

Basic Directory Scan

gobuster dir -u http://target.com -w /usr/share/wordlists/dirb/common.txt

Scans for directories using the provided wordlist.

Examples

Directory scan with common wordlist

Scan with file extensions

gobuster dir -u http://target.com -w wordlist.txt -x php,html,txt

Filter by status codes

gobuster dir -u http://target.com -w wordlist.txt -s 200,301

Hide 404 responses

gobuster dir -u http://target.com -w wordlist.txt -b 404

DNS subdomain enumeration

gobuster dns -d target.com -w subdomains.txt

Virtual host brute-forcing

gobuster vhost -u http://target.com -w vhosts.txt

Save output to file

gobuster dir -u http://target.com -w wordlist.txt -o gobuster-results.txt

Pro Tips

  • Use -t 50 or higher for faster scans (if the server can handle it).
  • Use -x to catch hidden files like index.php or config.txt.
  • Use -r to follow redirects and avoid false negatives.
  • Use -s and -b together to fine-tune output visibility.
  • Use -k when scanning HTTPS targets with self-signed certs.
